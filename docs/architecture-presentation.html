Camadas

Medallion Architecture

  • Bronze — CORE: customers, subscriptions, tickets, transactions, documents
  • Silver — MART/dbt intermediate: métricas por cliente, staging
  • Gold — MART/AI: CUSTOMER_360, EXECUTIVE_KPIS, churn scores, recomendações

Pipeline

Nativo + dbt

  • Dynamic Tables (refresh automático, nativo Snowflake)
  • dbt Core: 16 models — staging → intermediate → marts
  • Snowpark ML: churn, forecast, anomaly, recommendation

Multi-tenancy

org_id + Row Access Policy

  • org_id presente em toda tabela de dados
  • CONFIG.ORG_USER_MAP mapeia usuário → organização
  • GOVERNANCE.RAP_ORG_ISOLATION bloqueia linhas de outra org em runtime
  • Sem custom code por cliente — isolamento é estrutural, nativo do Snowflake
03

Como foi construído

Multi-agente Claude Code + agentcode

Processo

Orquestração de especialistas

Em vez de um assistente genérico, o framework agentcode roteia cada parte do trabalho para o agente do domínio certo — cada um entrando na etapa certa do fluxo design → implementação → revisão.

Agentes usados

  • snowflake-data-engineer — schemas, Dynamic Tables, RBAC
  • snowflake-cortex-expert — Cortex Agents, Semantic Models, Search
  • snowflake-governance-expert — RAP, masking, multi-tenancy
  • dbt-specialist — staging → intermediate → marts
  • security-reviewer / code-reviewer — revisão antes de cada merge

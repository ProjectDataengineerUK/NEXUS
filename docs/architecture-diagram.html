Multi-tenancy real — toda tabela carrega org_id; CONFIG.ORG_USER_MAP mapeia usuário → organização; GOVERNANCE.RAP_ORG_ISOLATION (Row Access Policy) bloqueia linhas de outra organização em runtime. Isolamento é estrutural, nativo do Snowflake — sem custom code por cliente.
Fora do Native App — do lado do provider (não dentro do Snowflake do cliente): Airflow (DAGs de ingestão Salesforce/Zendesk/Stripe/SAP/Oracle/HubSpot), Terraform (6 módulos: databases, warehouses, rbac, security, monitoring, app) e GitHub Actions (lint → test → deploy-dev → native-app-dev → release por tag).
